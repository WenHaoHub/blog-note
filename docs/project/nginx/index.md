::: tip
正向代理（Forward Proxy）和反向代理（Reverse Proxy）是在网络架构中使用 Nginx 服务器时常见的两种代理模式，它们的主要区别在于代理的对象和用途.
总结来说，正向代理的客户端是内部网络的用户，目的是让这些用户能够访问外部网络；而反向代理的客户端是外部网络的用户，目的是控制和优化对内部服务器的访问。
:::

## 正向代理：

1. 客户端配置代理服务器，当客户端请求网络资源时，由正向代理服务器代替客户端去获取资源，然后再将结果返回给客户端。
2. 正向代理隐藏了真实的客户端信息，因为服务器只能得到代理服务器的 IP。
3. 正向代理主要用于客户端访问限制网络的情况，如跨地域访问或访问受限资源。

## 反向代理：

1. 反向代理位于服务器端，当外部请求访问服务器资源时，由反向代理服务器接收请求，然后将请求转发到内部网络上的具体服务器。
2. 反向代理隐藏了真实的服务器信息，外部客户端不知道后端的服务器具体是谁。
3. 反向代理主要用于负载均衡、缓存静态内容、SSL 终端和提高安全性等。
